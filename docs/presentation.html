The same workflow, reimplemented everywhere

  • Every repo duplicates secrets, scripts, and GitHub Actions
  • Integrations drift — some repos handle reopened issues, others don't
  • One change means touching every repo individually

No single source of truth · inconsistent behaviour across the org

One YAML file = one automation


name: dependabot-automerge
pipeline:
  - given:
      trigger: github
      repos:
        - mongodb/mongodb-atlas-cli
        - mongodb/mongodb-atlas-local
        - mongodb/apix-action
    when:
      - event: pull_request
        action: opened
        actor: dependabot[bot]
    then:
      - github.approve_pr:
          owner: "{payload.repository.owner.login}"
          repo:  "{payload.repository.name}"
          number: "{payload.pull_request.number}"
      - github.enable_auto_merge:
          strategy: squash

How it works

GitHub webhook
→
Kanopy
→
Argo Events
→
automata
→
GitHub / Jira
  • Argo Events validates the HMAC signature and routes the event
  • automata matches it against every YAML automation and runs the steps
  • Built-ins: github.*, jira.*, builtin.jq

Onboarding a new repo

  • Install github.com/apps/apix-bot on the repo
  • Open a PR adding the repo to deploy/eventsource.yaml
  • Add the repo to the relevant automations/*.yaml files

The webhook is registered automatically on deploy — nothing else needed
